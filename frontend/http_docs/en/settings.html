Settings help

General settings

Changing settings described below requires manual app restart to apply them.

Theme (default System) - a general theme of the WASTT's appearance. It currently supported theme of the app:

  • System
  • Dark
  • Light

Language (default English) - currently supported languages:

  • English
  • German
  • Polish

Proxy settings

Settings described here involve proxy process restart for changes to apply.

Address & port (default localhost:8082) - IP address or hostname with the port on which mitmdump proxy should run. The same settings are applied to built-in browser.

Reload proxy - In case of emergency, there is an option to reload proxy without closing the app with ongoing testing project. It has additional option to retain the current Web Request Interceptor's scope while reloading the proxy.

Proxy logging (default OFF) - Posibility to log process of mitmdump and WASTT's custom proxy script if needed. After enabling this option, the log files titled proxy-DATE.log and mitmdump-DATE.log are saved to default log files location.

Proxy terminal (default OFF) - Option to open mitmdump proxy process in a new command line shell. If enabled, output from the proxy script will be displayed in the new terminal window and not in the standard python output to the console.
Due to app's current limitations, killing proxy process only terminates process in the terminal window, and not the terminal window itself.

Browser settings

Settings described here involve web browser process restart for changes to apply.

Browser types (default Chrome) - there are currently three supported variants of built-in browser to run. Availability of each of these variants depends depends on their respective binary being already installed in the system.
Currently supported variants:

  • Google Chrome
  • Microsoft Edge
  • Firefox

Disable infobars and notifications (default ON) - an option to disable all infobars, notices and notificatinos within the browser. There might be pop-ups informing about testing purposees of the built-in Selenium web driver browser. It is recommended to disable them for clearer visibility of pages and better acessibility during testing processes.

Disable certificate errors (default ON) - an option to fully disable missing certificate error and warnings in case of missing certificates on the web-pages. Disabling these warnings allow to directly open HTTPs pages.

Logs settings

Logs save location (default project's_dir/logs) - a local directory in which log files are saved to. Other modules create respective sub-directories (http_traffic, proxy, intruder, etc...) in directory specificed by this setting.

Logging HTTP Traffic (default OFF) - an option to log all HTTP flow that goes through WASTT's proxy. Including all requests' and responses' content that can be seen in HTTP Traffic sub-tab of Proxy tab.

Logging Request Interceptor (default OFF) - an option to log content of all HTTP(s) requests intercepted by Web Request Interceptor while it is turned on.

Debug settings

These settings are purely for testing and debugging purposes. Having them on and using during normal testing, may cause unwanted behaviour of the app and fail testing process.

Debug mode (default OFF) - it shows all application operations in the Python's output. It also adds few additional buttons in the GUI to test out its functionality.

Show running config (default OFF) - an option to show currently running configuration on the app's start up in the Python's output.
